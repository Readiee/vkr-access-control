# Инструкции для агента (Claude Code)

Краткая памятка для каждой сессии. Полный контекст — в корневом `CLAUDE.md` и `docs/PROJECT_BIBLE.md`.

## Порядок действий в новой сессии

1. Читать `docs/PROJECT_BIBLE.md` (раздел 6 — текущая фаза, раздел 7 — лог решений).
2. По типу задачи подгружать сателлиты (`docs/SAT_*.md`), см. §8.2.
3. Работать строго в рамках текущей фазы.

## Ключевые правила (из корневого CLAUDE.md)

- Не утверждать непроверенного. Гипотеза ≠ факт.
- Не делать коммиты самостоятельно.
- Не упрощать задачу.
- Каждое решение обосновывать (литература или техническое ограничение).
- Оценивать уверенность Green / Yellow / Red. При неопределённости >0.1 — уточняющие вопросы.
- Стиль текста — см. раздел «Правила написания текста» в корневом CLAUDE.md.

## Текущая фаза

Фаза 2 (старт 21.04, плановое окно 25.04–02.05): доработка кода по §3.6.7 PROJECT_BIBLE (три блока, FIX1–FIX15).

## Обязательно при правке `code/`

Загружать [docs/SAT_CODE_GUIDELINES.md](../docs/SAT_CODE_GUIDELINES.md) как контекст перед любой задачей фазы 2. Это не «рекомендации», это стандарт: архитектурные запреты, reasoning-дисциплина, правила owlready2, стиль комментариев, тесты (≥60%/≥80% Core), DoD.
