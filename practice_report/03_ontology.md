# 3. Адаптация онтологической модели под программный комплекс

## 3.1. Исходная онтология

Начальный вариант онтологии получен из научно-исследовательской работы, предшествовавшей практике. Он включал базовые классы предметной области (`Course`, `LearningModule`, `LearningActivity`, `Student`, `Competency`), концепты управления доступом (`AccessPolicy`, `Permission`) и каталог из четырёх SWRL-шаблонов: завершение, оценка, просмотр, компетенция с иерархией.

Исходная модель не покрывала ряд требований программного комплекса, выявленных в разделах 1.4 и 2.2:

- временные окна без поддержки текущего времени в SWRL;
- агрегатные условия (среднее, сумма, количество) при отсутствии агрегатов в Horn-логике;
- групповые ограничения с иерархией групп;
- логические комбинации условий (AND, OR) в виде составных политик;
- консолидация результата вывода в одном свойстве `is_available_for`.

Без устранения этих пробелов SWRL-каталог покрывает четыре типа из девяти, заявленных в составе ядра.

## 3.2. Расширения, выполненные в рамках практики

### 3.2.1. Класс `CurrentTime` и тип `date_restricted`

Спецификация SWRL не определяет функцию `now()`. Для типа `date_restricted` введён вспомогательный класс `CurrentTime` с datatype-свойством `current_dateTime`. Перед запуском логического вывода единственный экземпляр класса обновляется значением системного времени (предобработка ABox в Python). SWRL-правило сравнивает интервал политики с этим значением через стандартные built-ins.

### 3.2.2. Класс `AggregateFact` и тип `aggregate_required`

Агрегатные функции (`AVG`, `SUM`, `COUNT`) в SWRL отсутствуют. Введён класс `AggregateFact` с object property `for_student`, `for_policy` и datatype property `aggregate_value`. Для каждой пары (обучающийся, политика типа `aggregate_required`) предобработка вычисляет значение агрегата по перечню целевых элементов и материализует факт в ABox. SWRL-правило сравнивает значение факта с порогом политики.

### 3.2.3. Групповые ограничения

Добавлены object property `belongs_to_group` (обучающийся → группа) и `is_subgroup_of` с пометкой transitive (группа → надгруппа). Транзитивность даёт автоматическое распространение членства по иерархии: если группа `Beginners-A` подгруппа `Beginners`, обучающийся из `Beginners-A` удовлетворяет правилу, написанному на `Beginners`. Тип `group_restricted` оформлен SWRL-правилом, проверяющим достижимость целевой группы по `belongs_to_group ∘ is_subgroup_of*`.

### 3.2.4. Композитные политики

Логические комбинации условий потребовали двухуровневой семантики SWRL. На нижнем уровне расположены атомарные правила, выводящие предикат `satisfies(Student, Policy)`. На верхнем — мета-правило: если все подполитики составной политики удовлетворены, выводится `satisfies` для самой составной политики. Связь составной политики с её подполитиками задана object property `has_subpolicy`.

Конъюнкция (`and_combination`) реализована шаблонами арности 2 и 3 с явным `DifferentFrom` между подполитиками. Дизъюнкция (`or_combination`) реализована несколькими копиями правила, каждая из которых проверяет одну подполитику. Финальное правило H-1 объединяет все источники `satisfies` в `is_available_for`. Дополнительное правило H-2 покрывает зеркальный случай вывода через `viewed_via_completed`.

### 3.2.5. Сводный объём TBox

Итог расширения: 19 классов, 23 object properties, 14 datatype properties. SWRL-каталог содержит семь атомарных шаблонов (по одному на тип), шаблоны конъюнкции арности 2 и 3, шаблон дизъюнкции, мета-правило `has_subpolicy`, два хедер-правила (H-1, H-2) и вспомогательное правило `viewed_via_completed`. Полная диаграмма классов TBox приведена в Приложении В.

## 3.3. Компенсация ограничений OWL+SWRL

Часть требований не покрывается на уровне формализма и потребовала архитектурных приёмов, локализованных вне онтологии.

| Ограничение | Природа | Компенсация |
|---|---|---|
| Нет `now()` в SWRL | Статический снимок ABox | Предобработка: класс `CurrentTime` |
| Нет агрегатов | Horn-логика работает с индивидами, не с множествами | Предобработка: класс `AggregateFact` |
| Нет NAF при OWA | Открытый мир: отсутствие факта не означает его ложность | Постобработка: запрет по умолчанию (default-deny) на уровне приложения |
| Монотонность OWL | Резонеры не поддерживают TMS | Полный пересчёт: очистка выведенных фактов и повторный запуск логического вывода |

Итоговая схема обработки правила доступа: предобработка ABox (инжекция текущего времени и агрегатов) → запуск Pellet с TBox + SWRL → постобработка результата с применением запрета по умолчанию. Пара точек расширения изолирована в одном модуле и не затрагивает онтологию.
