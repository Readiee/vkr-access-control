# Заключение

## Результаты практики

В рамках производственной практики реализован прототип программного комплекса для управления правилами доступа к учебному контенту в СДО. Все восемь задач индивидуального задания выполнены; результаты по этапам сведены ниже.

| Этап индивидуального задания | Раздел отчёта | Ключевой результат |
|---|---|---|
| Анализ требований | 1 | Сформирован перечень из восьми групп функциональных и двенадцати нефункциональных требований; описаны границы системы и десять сценариев использования |
| Поддерживаемые типы правил | 2 | Определён состав ядра из девяти типов правил, обоснованы исключения, выделены три верифицируемых свойства корректности |
| Адаптация онтологии | 3 | Исходный TBox расширен пятью классами и шестью свойствами; SWRL-каталог дополнен композитными шаблонами и правилом восходящего вывода |
| Архитектура и интерфейсы | 4 | Спроектированы четыре компонента, шесть REST-роутеров, схема данных из четырёх репозиториев и двадцати Pydantic-моделей |
| Реализация серверной части | 5 | Реализован конвейер логического вывода и три алгоритма верификации; backend содержит 11 сервисов, покрытие тестами 84% при 82% по ядру |
| Пользовательский интерфейс | 6 | Реализованы три страницы: редактор правил под все типы политик, симулятор обучающегося и отчёт о верификации курса |
| Интеграционное тестирование | 7 | Пройдены 117 автоматизированных тестов и восемь демонстрационных сценариев; устранены девять обнаруженных дефектов |
| Подготовка отчётной документации | — | Подготовлен текст отчёта объёмом до 20 страниц с приложениями |

## Перспективы развития

За рамками практики оставлены направления, не входящие в основной контур управления правилами доступа к контенту курса.

- Расширение верификационного слоя свойствами оптимизационного класса: обнаружение избыточных правил и поглощения подмножеств правил более общими.
- Поддержка межкурсовых зависимостей и адаптивной маршрутизации по диапазонам оценок.
- Инкрементальный логический вывод вместо полного пересчёта выведенных фактов после каждого изменения ABox.
- Дополнение хедер-правила H-2 случаями составных триггеров и расширение шаблонов конъюнкции на арность выше трёх.

## Связь с ВКР

Этапы практики покрывают пять из шести технических задач выпускной квалификационной работы. Полученный прототип — рабочая основа для экспериментальной части ВКР: валидации корректности логического вывода на расширенном корпусе сценариев, оценки полноты обнаружения дефектов и измерения производительности при разных объёмах правил и фактов.
