# 6. Разработка пользовательского интерфейса

## 6.1. Состав интерфейса

Веб-интерфейс реализован на Vue.js с использованием библиотеки компонентов PrimeVue. Состояние клиента централизовано в Pinia-сторах (онтология и песочница). Доступ к серверу инкапсулирован в семи API-модулях, по одному на функциональный блок (курсы, политики, доступ, верификация, песочница, онтология, общий клиент).

Интерфейс разделён на три страницы.

| Страница | Назначение |
|---|---|
| Dashboard | Список курсов, навигация по дереву модулей и активностей, переход к редактору правил |
| Sandbox | Песочница методиста: тестовый обучающийся, изменение фактов, карта доступов с объяснением блокировок |
| VerificationReport | Отчёт о верификации курса с разбивкой по СВ-1, СВ-2, СВ-3 и обоснованием блокировок |

## 6.2. Редактор правил

Форма редактора динамически адаптируется под выбранный тип политики. Дискриминированная валидация на стороне сервера сопровождается клиентской проверкой обязательных полей. Композитные политики (`and_combination`, `or_combination`) редактируются в `CompositePolicyEditor`: подполитики добавляются через выпадающий список зарегистрированных типов, каждая подполитика разворачивается в собственную форму. Карточка политики (`PolicyRuleCard`) показывает читаемое описание правила, сформированное по шаблону без терминов SWRL и DL: «открыто после завершения теста 1», «средний балл по модулю не ниже 70».

## 6.3. Симулятор обучающегося

Песочница реализует UC-7a, UC-7b и UC-7c. `SandboxInspector` показывает карту доступов: дерево курса с отметкой «доступно» или «заблокировано» для текущего тестового обучающегося. Изменение фактов (отметка завершения, выставление оценки, смена группы, присвоение компетенции) вызывает пересчёт карты на сервере и обновление визуализации. При выборе заблокированного элемента раскрывается компонент `BlockingExplanation` со списком невыполненных условий.

## 6.4. Отчёт о верификации

Страница `VerificationReport` отображает результаты полной верификации курса. Для свойства СВ-1 выводятся пары конфликтующих политик с обоснованием вывода в виде дерева (`JustificationTreeView`); для СВ-2 — список циклов с перечнем элементов и типами рёбер; для СВ-3 — недостижимые элементы с указанием невыполнимого условия. Формулировки в отчёте приведены на языке методиста, без жаргона DL и SWRL.

Скриншоты ключевых экранов (редактор политики, песочница, отчёт о верификации) приведены в Приложении Е.
