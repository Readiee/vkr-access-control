# pz/figures/ — изображения для ПЗ

Здесь хранятся все растровые иллюстрации, попадающие в финальный DOCX. Источники изображений разнородны: экспорт из `diagrams/*/` (C4, UML), скриншоты интерфейса, графики результатов экспериментов, схемы, сделанные вручную.

## Правила именования

Плоская структура без подпапок. Имя файла:

```
<глава>-<номер-в-главе>-<короткое-имя>.<ext>
```

- `<глава>` — одна цифра (1, 2, 3, 4). Для изображений из приложений: `А`, `Б`, `В`, … (в соответствии с буквой приложения по ГОСТ, без Ё, З, Й, О, Ч, Ь, Ы, Ъ).
- `<номер-в-главе>` — порядковый номер рисунка в главе, соответствует нумерации по §8 SAT_PZ_REQUIREMENTS: «Рисунок 2.3» → `2-3-...`.
- `<короткое-имя>` — latin-only, kebab-case, максимум 40 символов.
- `<ext>` — `png` (предпочтительно, для растра) или `svg` (если вектор и нужен масштаб).

Примеры:

```
1-1-activity-as-is.png
2-1-c4-context.png
2-2-c4-container.png
2-5-sequence-policy-create.png
3-2-frontend-policy-editor.png
4-2-exp-reasoner-performance.png
А-1-swrl-rule-template.png
Б-1-tbox-overview.png
```

## Соответствие нумерации рисунка в тексте

Имя файла **всегда** соответствует номеру рисунка в тексте ПЗ. Если рисунок «Рисунок 2.3 – Архитектура» в тексте, файл — `2-3-architecture.png`. При перестановке рисунков в главе имена файлов переименовываются одновременно с правкой ссылок.

## Источники изображений

| Источник | Куда ложится | Примечание |
|---|---|---|
| Экспорт из `diagrams/c4/exports/pz/` | `pz/figures/` после ручной отрисовки в draw.io | Референс из PNG/SVG только подсказка, итог — draw.io |
| Экспорт из `diagrams/uml/exports/pz/` | То же | То же |
| Скриншоты UI из запущенного приложения | `pz/figures/` | ≥2× плотность (200+ dpi), светлая тема |
| Графики результатов экспериментов | `pz/figures/` | Генерируются скриптом из `experiments/`; источник скрипта коммитится |
| Ручные схемы (Miro/Excalidraw/draw.io) | `pz/figures/` | Оригинал сохраняется рядом с тем же именем и расширением `.drawio` (или в `diagrams/<type>/`) для редактирования |

## Ограничения по качеству

- Минимальная плотность: 200 dpi для экспортов. Меньше — нечитаемо при печати.
- Растровые PNG без фона нежелательны: Word иначе показывает серый прямоугольник.
- Размер в пикселях: не шире 2400 px по длинной стороне (иначе DOCX раздувается). Обычно 1600–2000 px достаточно.
- SVG допустим, если draw.io или источник вектора стабильно рендерит в Word. Иначе — PNG.

## Что сюда не кладём

- Исходники DSL (Structurizr, PlantUML, Mermaid) — в `diagrams/`.
- Промежуточные рендеры (референсы для draw.io) — в `diagrams/<type>/exports/pz/`.
- Файлы `.drawio` (если их несколько) — в `diagrams/<type>/pz/`. Если схема одиночная и уникальна для этой картинки — допустимо положить `.drawio` рядом с PNG в `pz/figures/` для удобства.

## Финальная сборка DOCX

При переносе в Word картинки вставляются из `pz/figures/` по имени. Подписи — по правилам §8 SAT_PZ_REQUIREMENTS: «Рисунок N.M – Название» снизу по центру, без точки в конце.
