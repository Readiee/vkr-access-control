## Система нумерации источников

Финальная нумерация — по первому появлению в тексте, арабскими цифрами в квадратных скобках `[N]` (§13.1 Требований). В MD-черновике ручная нумерация ломается при любом переписывании абзаца. Используется система плейсхолдеров, описанная ниже.

### Формат ссылки в тексте глав

В любом файле `pz/00_introduction.md` … `pz/07_appendices.md` ссылка на источник пишется как `[@TAG]`, где TAG — идентификатор из §1.8 PROJECT_BIBLE (Z1–Z10, P1–P31, N1–N16).

```
Finin et al. [@P3] формализовали RBAC в OWL.
Nazyrova et al. [@Z7] применили онтологический подход к проверке учебных планов.
Систематический обзор моделей управления доступом [@N11] охватывает 98 подходов.
```

Для читаемости черновика допустима расширенная форма `[@TAG-мнемоника]`. Всё после `-` при сборке игнорируется:

```
Finin et al. [@P3-Finin2008] формализовали RBAC в OWL.
Kolovski et al. [@P21-Kolovski2007] применили DL-reasoning к XACML.
```

### Разрешённые формы

| Форма | Когда | Пример в черновике |
|---|---|---|
| `[@TAG]` | Основная форма ссылки | `по данным [@P3]` |
| `[@TAG-мнемоника]` | С подсказкой для читаемости черновика | `Finin et al. [@P3-Finin2008]` |
| `[@TAG, с. 12]` | Со страницей (§4.9 Требований) | `формула [@P17, с. 12]` |
| `[@TAG1; @TAG2]` | Групповая | `[@P3; @P4; @P5]` |
| Жёсткая `[5]` | ❌ запрещена до финальной сборки | — |

### Реестр тегов

Основной реестр — [§1.8 PROJECT_BIBLE](../docs/PROJECT_BIBLE.md). Серии:

- **Z1–Z10** — источники из задания ИСУ.
- **P1–P31** — пул фазы 0 (литобзор).
- **N1–N16** — источники, выявленные на фазах 1 и Deep Research.

Если нужен источник, которого нет в §1.8:

1. Добавить запись в §1.8 PROJECT_BIBLE с новым номером в серии N (N17, N18, …). §1.8 — единственный реестр, все теги рождаются там.
2. Внести библиографическую запись по ГОСТ в блок «Заготовки записей» ниже.
3. Использовать новый тег `[@Nxx]` в тексте глав.

**Запрещено:** вводить тег, которого нет в §1.8 PROJECT_BIBLE. Иначе при сборке DOCX скрипт или пропустит источник, или присвоит дубликат номера.

### Сборка финального списка (MD → DOCX)

Один раз, когда все главы готовы:

1. Пройти по файлам `pz/00..07` в порядке нумерации, построчно.
2. Собрать все `[@TAG]` и `[@TAG-…]` в порядке первого появления.
3. Присвоить порядковые арабские номера `[1]`, `[2]`, …
4. Заменить в тексте глав `[@TAG]` → `[N]`, `[@TAG-…]` → `[N]`.
5. Сформировать итоговый «СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ» в порядке присвоения, беря библиографические записи из блока «Заготовки записей» ниже.

Скрипт сборки — артефакт фазы 4. Пишется при финализации первой готовой главы (чтобы обкатать на реальных данных). Алгоритм несложный: регулярка `\[@([A-Z]\d+)(?:-[^\]]+)?(?:,[^\]]+)?\]` + словарь TAG → номер. Должен быть идемпотентным и безопасным при повторных прогонах до сборки.

### Правила для электронных источников

- URL без гиперссылки (удалить синий подчёркнутый стиль в Word).
- «Дата обращения» обязательна для любого веб-ресурса.
- Для DOI вставлять сам DOI, не полный URL через doi.org.
