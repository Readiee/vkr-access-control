### 4.5 Глава 4. Экспериментальная оценка

> **Статус (24.04):** все три эксперимента закрыты, результаты — в [pz/figures/exp1/](../pz/figures/exp1/), [pz/figures/exp3/](../pz/figures/exp3/), [pz/figures/exp4/](../pz/figures/exp4/). Глава пишется по готовой фактуре.
> **Структура (решение 25.04 §7):** один подраздел на эксперимент, сплошным текстом по ритму постановка → методика → результаты → обсуждение. Без 4-уровневой разбивки внутри эксперимента — это убирает бюрократический оверкилл и снижает риск воды.
> **План §5.1 PROJECT_BIBLE (редакция 24.04):** три эксперимента вместо семи. EXP5 (сравнение с Moodle) → глава 1 §1.3; EXP6 (демо happy_path) → глава 3 §3.7.3 + Приложение В; EXP7 (маппинг на 5 СДО) → Приложение Ж. Архитектура интеграции — глава 2 §2.6 + глава 3 §3.4.
> **Граница с тестированием в §3.7:** §3.7 проверяет корректность реализации относительно спецификации (unit + integration, бинарный pass/fail + покрытие). Эксперименты главы 4 проверяют эффективность алгоритмов на выборке с метрикой Precision/Recall/Accuracy.

## 8. Рисунки (§4.4 Требований)

- Размещаются **под текстом** с первой ссылкой (или на следующей странице, если не помещается).
- Нумерация **по главам**: `Рисунок 2.3` = третий рисунок второй главы.
- **Подпись снизу по центру**, формат:

  ```
  Рисунок 2.3 – Архитектура разработанной системы
  ```

- После подписи — **без точки**. Наименование — с прописной буквы.
- Многострочная подпись — одинарный межстрочный интервал.
- **На все рисунки обязательны ссылки в тексте**: «как показано на рисунке 2.3», «(рисунок 2.3)».
- **Диаграммы C4/UML: финальная отрисовка в draw.io** по референсу из `diagrams/<type>/exports/pz/` (см. §3.5.2 PROJECT_BIBLE).

## 9. Таблицы (§4.5 Требований)

- Размещаются **под текстом** с первой ссылкой.
- Нумерация **по главам**: `Таблица 1.2`.
- **Название сверху слева, без абзацного отступа**, формат:

  ```
  Таблица 1.2 – Сравнение механизмов управления доступом в СДО
  ```

- После названия — без точки.
- **Ссылки в тексте обязательны**: «см. таблицу 1.2».
- **Нельзя вставлять таблицу как изображение** — всегда пересобирать в редакторе.
- **Перенос таблицы**: после шапки вставить строку с номерами столбцов (1, 2, 3…); над продолжением написать `Продолжение таблицы 1.2`.

## 10. Формулы (§4.6 Требований)

- **Отдельной строкой**, с абзацного отступа, выравнивание по центру (допустимо — по левому краю).
- Выше и ниже формулы — **не менее одной свободной строки**.
- Нумерация **по главам**: `(2.1)` = первая формула второй главы.
- Номер — **в круглых скобках справа у правого поля**, на уровне формулы.
- **Ссылки в тексте**: «по формуле (2.1)», «из уравнения (2.1) выразим…».
- **Пояснения** непосредственно под формулой, первая строка пояснения — `где` (без двоеточия).
- После формулы в теле расшифровки — **запятая**; в конце предложения — точка.
- Единицы измерения — **либо международные, либо русские, не смешивать** (ГОСТ 8.417-2002).
- **Неразрывный пробел** перед единицами измерения.

Пример:

```
Время верификации курса оценивается суммой времени reasoning и времени графового анализа

                    T_verify = T_reason + T_graph,                 (2.1)

где T_reason — время работы DL-резонера, мс;
    T_graph  — время обхода графа зависимостей, мс.
```

**Набор формул:** MathType, Microsoft Equation или встроенный редактор Word.

В ВКР **не допускаются**:

- обороты разговорной речи, техницизмы, профессионализмы;
- разные термины-синонимы для одного понятия *(см. [SAT_PZ_STYLE.md §16](SAT_PZ_STYLE.md) — терминологический словарь)*;
- иностранные слова при наличии равнозначных русских;
- произвольные словообразования;
- сокращения слов, кроме установленных правилами орфографии и стандартами;
- сокращение обозначений единиц физических величин, если они употребляются без цифр (кроме головок/боковиков таблиц и расшифровок в формулах/рисунках).

## 18. Маршрут «MD → DOCX»

1. Главы пишутся в Markdown в папке [pz/](../pz/).
2. Финализация каждой главы: прогон чек-листа [SAT_PZ_STYLE.md §11](SAT_PZ_STYLE.md) и чек-листа §19 ниже.
3. Перенос в [docs/thesis-template.docx](thesis-template.docx) вручную: нумерация рисунков/таблиц, отступы заголовков, стили Word.
4. Финальная вёрстка: номера страниц, содержание с отточиями, стили заголовков, перенос формул из Markdown в Equation Editor.
5. Сборка PDF: Word → экспорт PDF (не печать) с вложением шрифтов.
6. Антиплагиат и AI-детектор — ручная проверка.

## 19. Чек-лист сдачи фрагмента в DOCX

Прогонять перед сборкой финальной версии:

- [ ] Заголовки структурных элементов — ПРОПИСНЫМИ, по центру, без точки, без нумерации, с новой страницы.
- [ ] Заголовки глав — с новой страницы, полужирный, без точки, без переносов, без аббревиатур.
- [ ] Подразделы и подпункты — не разрываются на страницы.
- [ ] Абзацный отступ 1,25 см везде одинаковый.
- [ ] Шрифт TNR 14, интервал 1,5, поля 30/15/20/20 мм.
- [ ] Номера страниц — центр низа, арабские, без точки, включая приложения.
- [ ] Рисунки: подпись снизу по центру «Рисунок N.M – Название» без точки.
- [ ] Таблицы: название сверху слева «Таблица N.M – Название» без точки; ни одной таблицы-картинки.
- [ ] Формулы: отдельной строкой, номер справа `(N.M)`, пояснения через «где».
- [ ] Единицы измерения: все русские или все международные (не смешиваются).
- [ ] Кавычки «ёлочки», без латинских `" "`.
- [ ] Тире: только длинное (—) в тексте и короткое (–) в диапазонах; дефис (-) — внутри слов.
- [ ] Ё — либо везде, либо нигде.
- [ ] Ссылки на рисунки, таблицы, формулы, приложения, источники — минимум одна на каждый элемент.
- [ ] Список источников — по порядку появления, с абзацного отступа, с точкой после номера.
- [ ] Все URL — с датой обращения, без гиперссылки.
- [ ] Содержание — отточие к номеру страницы, заголовки дословно.
- [ ] Приложения — с новой страницы, заголовок полужирным симметрично.
- [ ] Список сокращений — столбец без пунктуации, слева аббревиатура, справа через тире.
- [ ] Нет запрещённых выражений из [SAT_PZ_STYLE.md Приложение А](SAT_PZ_STYLE.md).
- [ ] Антиплагиат: ≥80 % оригинальности (перед сдачей в ИСУ).
- [ ] AI-детектор: ручная проверка, вручную переписаны все флаги.

## 20. Приоритеты и ориентиры по объёму

> **Назначение:** распределить уровень детализации по подразделам ПЗ. Позволяет писать осознанно: где детально с примерами и листингами, где сплошным текстом без раскопок, где обзорно с отсылкой к артефактам. Цель — избежать «ровного слоя текста» с водой.

### 20.1 Принципы распределения

- **Акцент** даётся на разделы, прямо связанные с замечаниями комиссии первого рассмотрения (§1.4 PROJECT_BIBLE) и с положениями на защиту (§3.3 PROJECT_BIBLE).
- **Стандартное изложение** — на разделы постановки задачи, описания внутренней структуры, обзорные материалы.
- **Обзорный режим** — на разделы, признанные не научным вкладом (UI, развёртывание, фронтенд) или хорошо знакомые читателю (стандартные технологии, общеизвестные термины).

- На каждые 200 слов как минимум одно конкретное число, дата, фамилия с годом или название источника (см. [SAT_PZ_STYLE.md §7.2](SAT_PZ_STYLE.md)).
